04 — Le moteur de déclin

Une seule formule régit toute la mémoire.

strength = 0.5 ^ ( elapsed / halfLife )
À t=0, la force est 1,0.
Après une demi-vie, 0,5.
Asymptotique — pas de falaises d'expiration abruptes.
decay.ts 
export function computeStrength(referenceAt: string, halfLifeDays: number, now = Date.now()): number {
  const elapsedDays = (now - new Date(referenceAt).getTime()) / MS_PER_DAY;
  return Math.pow(0.5, elapsedDays / halfLifeDays);  // injectable pour tests déterministes
}

export function computeEffectiveHalfLife(
  base: number, accessCount: number, isError: boolean,
  retrievalBoostDays = 2, errorMultiplier = 2
): number {
  const b = isError ? base * errorMultiplier : base;   // effet amygdale
  return b + accessCount * retrievalBoostDays;          // répétition espacée
}

04 — Entrée mémoire

Les niveaux de confiance
déterminent l'ordre d'injection.

Chaque entrée porte un niveau de confiance. Les agents pondèrent les mémoires en conséquence : les niveaux supérieurs l'emportent sur les niveaux inférieurs en cas de conflit.

memory.types.ts 
type ConfidenceTier =
  | 'verified'   // ← confiance maximale
  | 'observed'
  | 'inferred'
  | 'stale';    // ← jamais injectée

// Injection : verified → observed → inferred
// Les entrées périmées sont toujours exclues.
Vérifié
Vérité confirmée
Plusieurs agents ou exécutions ont validé cette observation. Priorité maximale dans l'injection de prompt — le plus respecté.
→ Injecté en premier
Observé
Fait de source unique
Enregistré une fois par un agent. Suffisamment fiable pour être affiché, mais pas encore corroboré. Deuxième priorité.
→ Injecté en deuxième
Inféré
Motif dérivé
Généré par la consolidation de similarité Jaccard. Un motif potentiel, non confirmé. Troisième priorité.
→ Injecté en dernier
Périmé
Invalidé par Git
Déclassé lorsque les relatedPaths intersectent des fichiers modifiés depuis la dernière consolidation. Les décisions annulées deviennent automatiquement périmées.
→ Jamais injecté

05 — Pipeline de consolidation

Les observations individuelles deviennent une sagesse partagée.

Déclenché via valora consolidate ou automatiquement après un pipeline feedback réussi. Quatre étapes, toujours en séquence.

ÉTAPE 01
Élaguer les entrées déclinées
Calculer computeStrength() pour chaque entrée. Supprimer les entrées en dessous du seuil d'élagage (0,05). Les stockages restent légers automatiquement — aucune maintenance manuelle.
strength < 0.05 → delete
→
ÉTAPE 02
Invalidation Git
Exécuter git log --name-only --since=lastConsolidatedAt. Les entrées dont les relatedPaths intersectent des fichiers modifiés sont déclassées en périmées.
paths ∩ changed → stale
→
ÉTAPE 03
Fusion par similarité de Jaccard
Analyser les entrées épisodiques par paires. Calculer |A∩B| / |A∪B| sur les ensembles de tags. Les paires avec un chevauchement ≥ 0,6 fusionnent en une entrée sémantique. Aucun embedding requis.
Jaccard ≥ 0.6 → semantic
→
ÉTAPE 04
Promotion automatique
Les entrées épisodiques vérifiées à accès élevé non captées par la fusion de similarité sont promues directement dans le stockage sémantique. Les motifs fréquemment interrogés méritent une demi-vie plus longue.
verified + high-access → semantic
