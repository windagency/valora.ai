04 — The Decay Engine

One formula governs all memory.

strength = 0.5 ^ ( elapsed / halfLife )
At t=0, strength is 1.0.
After one half-life, 0.5.
Asymptotic — no hard expiry cliffs.
decay.ts 
export function computeStrength(referenceAt: string, halfLifeDays: number, now = Date.now()): number {
  const elapsedDays = (now - new Date(referenceAt).getTime()) / MS_PER_DAY;
  return Math.pow(0.5, elapsedDays / halfLifeDays);  // now injectable for deterministic tests
}

export function computeEffectiveHalfLife(
  base: number, accessCount: number, isError: boolean,
  retrievalBoostDays = 2, errorMultiplier = 2
): number {
  const b = isError ? base * errorMultiplier : base;   // amygdala effect
  return b + accessCount * retrievalBoostDays;          // spaced repetition
}

04 — Memory Entry

Confidence tiers
drive injection order.

Every entry carries a confidence tier. Agents weight memories accordingly: higher tiers override lower ones when they conflict.

memory.types.ts 
type ConfidenceTier =
  | 'verified'   // ← highest trust
  | 'observed'
  | 'inferred'
  | 'stale';    // ← never injected

// Injection: verified → observed → inferred
// Stale entries are always excluded.
Verified
Confirmed truth
Multiple agents or runs have validated this observation. Highest priority in prompt injection — respected most.
→ Injected first
Observed
Single-source fact
Recorded once by one agent. Reliable enough to surface, but not yet corroborated. Second priority.
→ Injected second
Inferred
Derived pattern
Generated by Jaccard-similarity consolidation. A potential pattern, not confirmed. Third priority.
→ Injected last
Stale
Git-invalidated
Downgraded when relatedPaths intersect with files changed since last consolidation. Reverted decisions auto-stale.
→ Never injected

05 — Consolidation Pipeline

Individual observations become shared wisdom.

Triggered via valora consolidate or automatically after a successful feedback pipeline. Four steps, always in sequence.

STEP 01
Prune decayed entries
Compute computeStrength() for every entry. Remove entries below the prune threshold (0.05). Stores stay lean automatically — no manual housekeeping.
strength < 0.05 → delete
→
STEP 02
Git invalidation
Run git log --name-only --since=lastConsolidatedAt. Entries whose relatedPaths intersect changed files are downgraded to stale.
paths ∩ changed → stale
→
STEP 03
Jaccard similarity merge
Scan episodic entries pairwise. Compute |A∩B| / |A∪B| on tag sets. Pairs with overlap ≥ 0.6 merge into a semantic entry. No embeddings required.
Jaccard ≥ 0.6 → semantic
→
STEP 04
Auto-promote
High-access verified episodic entries not caught by similarity merge are promoted to the semantic store directly. Patterns agents repeatedly query deserve a longer half-life.
