Sequence Notes
Command Resolution
CommandResolver · 3-tier fallback
src/cli/command-resolver.ts
Tries requested provider first, then default, then any available. Validates model name before returning. Throws if no provider can satisfy
Coordinator · agent resolution
src/cli/execution-coordinator.ts
Static path reads data/agents/*.yaml. Dynamic path calls dynamicAgentResolver and may prompt the user if confidence < 50%
PipelineExecutor · stage scheduling
src/executor/pipeline.ts
Stages grouped as parallel or sequential. Parallel groups run via Promise.all(). Sequential via async reduce(). Variables pre-resolved before any stage starts
Tool Loop
Loop termination conditions
StageExecutor · max_tool_iterations default 20
Loop exits when LLM returns no tool_calls, or when max_tool_iterations is reached. On exhaustion, a forced final LLM call with no tools extracts JSON output
Error vs Guidance
tool-execution.service.ts · MAX_TOOL_FAILURES = 5
Results starting with “Error:” count toward MAX_TOOL_FAILURES (5). Plain strings are guidance — recoverable LLM mistakes like file-not-found that don’t trigger the hard stop
Message history accumulation
Each iteration appends to messages[]
Each iteration appends {role:’assistant’, tool_calls} then {role:’tool’, content} messages. Full history sent to LLM on every call — enabling multi-turn tool reasoning
Session & Caching
Session post-execution
src/session/
After execution, complete() or fail(error) triggers persist(true) — immediate, non-debounced flush. Token usage, worktree stats, and stage outputs all committed
Anthropic Provider · prompt caching
src/llm/providers/anthropic.provider.ts
Applies cache breakpoints to large system prompts. Token usage tracks prompt_tokens, completion_tokens, cache_creation_input_tokens, and cache_read_input_tokens per stage
ResultPresenter · context guard
src/cli/result-presenter.ts
Validates context influence ≥ 35% before displaying output. Ensures responses are grounded in actual session state, not hallucinated content
