State Reference
Active & Paused
active
create() or resume(id) from any prior status
Commands recorded, context accumulated. Persist debounced: 1s normally, 5s for rapid commands (<10s apart). A .snapshot.json is saved alongside every persist — last 3 commands + keys _loaderCache and _stageOutputs
paused
pause() → resume(id) → active
Soft suspend — currentSession retained in memory (unlike completed/failed which null it). Immediate persist. Resumes back to active without reloading from disk
Terminal States
completed
complete() → persist(true) → currentSession = null
Terminal but resumable. persist(true) fires immediately (no debounce). Token usage and worktree stats are collected. currentSession set to null afterwards
failed
fail(error) → context.error stored → persist
Error message stored in context.error before persist. resume(id, resetContext: true) clears the error key and resurrects the session to active
Retention Scheduler
compressed (.gz)
age > 30d → gzip in-place
File compressed with gzip. Status label unchanged in metadata. Still fully resumable — the session manager decompresses transparently on load
deleted
age > 90d or size > 50 MB or count > 100
Background cleanup runs every 24h. Rules applied in order: max-age (90d), then size limit (50 MB oldest-first), then count limit (100 oldest-first)
