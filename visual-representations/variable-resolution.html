Reference
Variable Scopes
$ARG_* — CLI Arguments
variables.ts:102 · resolveArg()
Normalised to four forms: positional index ($ARG_1), original Commander key, kebab-case, and snake_case. Missing args return "Not specified" rather than throwing.
$CONTEXT_* — Session
variables.ts:203 · resolveContext()
Only keys referenced in the pipeline are fetched from the session (extractPipelineContextReferences). Dot-notation traverses nested objects. Missing returns "Not specified".
$ENV_* — Environment
variables.ts:210 · resolveEnv()
Direct process.env lookup. In strict mode (default), an unresolved reference throws a ValidationError. No fallback chain — either present or error.
$STAGE_* — Stage Outputs
variables.ts:177 · resolveStage()
Returns null (not an error) for skipped stages — allows downstream conditionals to handle gracefully. Supports deep paths: $STAGE_review.issues.0.severity.
Resolution Phases
① Pre-Resolution
input-pre-resolver.ts:46 · preResolveStaticInputs()
Runs once before any stage executes. Stages without $STAGE_* refs are fully resolved and cached. ARG, CONTEXT, ENV are all known at this point. Avoids redundant re-resolution.
File Enrichment
input-pre-resolver.ts:147 · enrichWithFileContents()
Input keys ending with _file, _file_arg, or _path are read from disk. Content stored as key_content alongside the original path. Deduplicates reads via a path cache.
② Runtime Resolution
stage-executor.ts:627 · resolveStageInputs()
For deferred stages, inputs are resolved at execution time against the live VariableResolver — which by then has $STAGE_* from all completed upstream stages.
Conditional Evaluation
pipeline.ts:470 · evaluateConditional()
Stage conditionals are variable-resolved first, then cast to boolean. The string "true" passes. Skipped stages leave $STAGE_name as null — downstream stages see null, not an error.
Advanced Features
Regex Interpolation
variables.ts:47 · resolveString()
Pattern /\$([A-Z]+)_([a-zA-Z0-9_.-]+)/g replaces all matches in a single pass. Scope group dispatches to the appropriate resolver. valueToString() handles null, objects, arrays.
Deep Recursive Resolution
variables.ts:28 · resolve<T>()
Accepts strings, arrays, and nested objects — recursively walks the entire structure. An entire stage inputs record can be passed and every string leaf is interpolated.
Stage Output Recording
execution-context.ts:111 · recordStageCompletion()
Called by PipelineExecutor after each successful stage. Calls addStageOutputs() immediately — $STAGE_name.* becomes available to the next stage without waiting for the full pipeline to finish.
Cross-Command Persistence
command-executor.ts:785 · session._stageOutputs
All stage outputs are merged into session._stageOutputs after the command completes. The next valora command restores them as initialStageOutputs so $STAGE_* from prior commands remains accessible.
