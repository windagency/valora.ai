Component Reference
CLI Layer
CLI Entry
src/cli/index.ts
Commander.js arg parsing, first-run check, interactive command palette when no args given
CommandExecutor
src/cli/command-executor.ts
Validates prerequisites, creates/resumes session, tracks token usage, calls ResultPresenter
CommandResolver
src/cli/command-resolver.ts
Loads YAML from data/commands/. 3-tier provider fallback: requested → default → any available. Validates model name
SessionManager
src/session/
Creates or resumes sessions. Commands share state via $CONTEXT_key across stages and sessions
Pipeline & Execution
ExecutionCoordinator
src/cli/execution-coordinator.ts
Resolves effective agent (static config or dynamic resolver), creates ExecutionContext, selects strategy via factory
PipelineExecutor
src/executor/pipeline.ts
Pre-resolves $ARG_*, $ENV_*, $CONTEXT_* before any stage runs. Groups: sequential via reduce(), parallel via Promise.all()
StageExecutor
src/executor/stage-executor.ts
Loads prompt + agent defs, builds message history, drives the LLM ↔ tool loop until no tool_calls remain or iterations exhausted
ResultPresenter
src/cli/result-presenter.ts
Displays outputs, token usage, session ID. Validates context influence ≥ 35% — architectural guard ensuring grounded responses
LLM & Tools
LLM Providers
src/llm/providers/
Anthropic (prompt caching), OpenAI, Google, Cursor, XAI, Moonshot. Returns LLMCompletionResult with full token usage breakdown
Tool Execution
src/executor/tool-execution.service.ts
Dispatches built-in tools (write, read_file, grep, glob, run_terminal_cmd…) or MCP tools. Error results count toward limit of 5
Tool Loop
StageExecutor · max_tool_iterations default 20
Cycles LLM ↔ tools until finish_reason is stop. On exhaustion, a forced final call with no tools extracts JSON output
Hard Stop
MAX_TOOL_FAILURES = 5
Results prefixed “Error:” count toward the limit. Guidance results (plain strings) do not — they are recoverable LLM mistakes
